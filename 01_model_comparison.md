## 3. 평가 기준

### 평가 축

1. **근거 제시 능력**

   * 시장 규모 산출 시 수치 근거 제공 여부
   * 불확실한 정보에 대한 처리 방식

2. **결과물 활용성**

   * 사업계획서 활용 가능성
   * IR 자료 활용 가능성

3. **Hallucination 방지**

   * 존재하지 않는 데이터 생성 여부
   * 정보 부족 시 확인 질문 수행 여부

4. **형식 준수**

   * 지정된 출력 형식 준수 여부

5. **문맥 유지 및 수정 요청 대응력**

   * 국가 변경 시 기존 조건 유지
   * 타깃 변경 시 기존 분석 구조 유지
   * 추가 조건 반영 능력

## 6. 최종 선정 결과

### 선정 모델

**Claude Sonnet 4.6**

### 선정 사유

본 과업인 **멘탈케어 시장 분석(TAM, SAM, SOM)** 수행 결과, Claude를 최종 선정하였다.

Claude는 평가 항목인 **근거 제시 능력, 결과물 활용성, Hallucination 방지, 형식 준수, 문맥 유지 및 수정 요청 대응력**에서 모두 우수한 성능을 보여 총점 **5점**을 기록하였다.

특히 시장 규모 산출 과정에서 수치 근거와 출처를 함께 제시하였으며, 확인이 어려운 정보에 대해서는 **"확인 필요"** 또는 **"확인 불가"**를 명시하여 환각(Hallucination) 발생 가능성을 최소화하였다.

또한 추가 시장 정보 제공이나 수정 요청이 발생하더라도 기존 출력 형식을 유지하면서 TAM, SAM, SOM 분석 결과를 일관되게 반영하였다.

생성된 결과물은 실제 사업계획서, 시장조사 보고서, 투자자 발표 자료(IR)에 활용 가능한 수준으로 작성되어 실무 활용성이 가장 높게 평가되었다.

반면 GPT는 근거 제시와 형식 준수 측면에서는 우수하였으나, 일부 수정 요청 상황에서 기존 요구사항을 완전하게 반영하지 못하는 사례가 확인되었으며, 확인이 어려운 정보를 명확하게 표시하지 못한 경우가 있었다.

Gemini는 기본적인 형식은 준수하였으나 시장 규모 산출 근거와 세부 분석 수준이 부족하여 실무 활용성이 상대적으로 낮게 평가되었다.

# 7. 환각(Hallucination) 정의

본 프로젝트에서 환각(Hallucination)은 **실제 사실과 다른 정보를 사실처럼 생성하는 현상**으로 정의한다.

예시:

* 존재하지 않는 시장 규모 수치 생성
* 존재하지 않는 연구 보고서 인용
* 확인되지 않은 데이터를 사실로 단정
* 실제 출처가 없는 통계 수치 제시

반면 창의적 생성(Creative Generation)은 사용자가 요청한 범위 내에서 허구의 이야기, 아이디어, 예시를 생성하는 행위이며 사실 검증 대상이 아니므로 환각과 구분한다.

따라서 본 프로젝트에서는 시장 규모, 통계, 기업 정보, 정책 정보 등 사실 검증이 가능한 정보를 잘못 생성한 경우를 환각으로 판정하였다.

---

# 8. 환각 검증 기준

## PASS A

공식 통계, 시장조사 보고서 또는 신뢰 가능한 출처와 비교했을 때 주요 수치가 일치하거나 오차 범위 ±10% 이내인 경우

## PASS B

정확한 자료 확인이 어려운 경우 "확인 불가", "추가 검증 필요", "참고용 추정치" 등을 명시한 경우

## FAIL

다음 중 하나 이상 해당하는 경우

* 존재하지 않는 통계 생성
* 존재하지 않는 보고서 인용
* 확인되지 않은 수치를 사실처럼 단정
* 출처 없이 시장 규모를 제시
* 사용자가 요구한 제약 조건 무시

# 10. 심층 인터뷰

## Q1. 비용 제약으로 무료 모델만 사용해야 한다면 품질을 유지하기 위해 어떻게 개선할 것인가?

### 답변

무료 모델만 사용해야 하는 경우 프롬프트 구조를 강화하여 품질 저하를 최소화한다.

적용 방안

1. 입력 템플릿 표준화
2. 시장 범위, 국가, 수익 모델 필수 입력
3. 정보 부족 시 최대 3개 질문 강제
4. 근거 없는 수치 사용 금지
5. 확인 불가 항목 명시 의무화

이를 통해 모델 자체 성능 부족을 프롬프트 설계로 보완할 수 있다.

---

## Q2. 업무 정책(규정·수치)이 자주 변경된다면 환각을 줄이기 위한 운영 전략은?

### 답변

정책 및 수치가 자주 변경되는 업무에서는 모델 내부 지식에 의존하지 않고 최신 자료를 우선 활용해야 한다.

적용 방안

1. 최신 자료를 입력으로 제공
2. 정책 정보는 사용자 입력을 우선 적용
3. 출처 없는 수치 사용 금지
4. 확인 불가 시 추정 금지
5. 정책 변경 사항을 별도 입력 블록으로 관리

이를 통해 오래된 지식 기반 환각을 줄일 수 있다.

---

## Q3. 10턴 이상 대화에서 문맥이 끊기는 문제가 발생한다면 어떻게 개선할 것인가?

### 답변

장기 대화에서는 이전 조건이 누락될 위험이 존재한다.

개선 방안

1. 매 턴 현재 확정 조건 요약
2. 변경 조건만 별도 기록
3. 핵심 변수 고정 블록 유지
4. 시장 범위, 국가, 수익 모델 등 주요 조건 재출력
5. 결과 생성 전 조건 검증 단계 추가

예시

현재 유지 조건

* 서비스: 감정일기 기반 AI 멘탈케어 앱
* 시장 범위: 글로벌
* 수익 모델: 월 구독 9,900원
* 핵심 기능: CBT 기반 AI 챗봇

위와 같은 방식으로 조건을 반복 확인하면 문맥 손실을 최소화할 수 있다.


### 최종 결론

정확성, 재현성, 실무 활용성 및 Hallucination 방지 능력을 종합적으로 고려할 때, **Claude Sonnet 4.6이 멘탈케어 시장 분석(TAM, SAM, SOM) 업무에 가장 적합한 모델로 판단된다.**
